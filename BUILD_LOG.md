## Log

### 🎉 DEVICE WORKING END-TO-END (on hardware) — user-confirmed
- Flashed to the board, on WiFi `CommunityFibre10Gb_65587`, talking to the deployed gateway.
  Full voice loop confirmed: speak → Groq STT → Claude (per-user memory) → Kokoro TTS, audio
  plays back cleanly.
- Hardware bring-up fixes this session:
  - **Touch boot crash / "keeps turning off":** stock BSP hard-aborts if the FT3168 I2C probe
    fails at boot (intermittent → boot loops). Patched `bsp_display_indev_init` to retry + be
    non-fatal. Lives in git-ignored `managed_components/`, so preserved as
    **`firmware/patches/apply_bsp_patches.sh`** — re-run after any `idf.py reconfigure`.
  - **WiFi keeps dropping:** disabled WiFi power-save (`esp_wifi_set_ps(WIFI_PS_NONE)`) — modem-sleep
    stalled the audio stream and tore down the WS. Gateway `ping_timeout` 20→60 s.
  - **Weak signal at new location:** added a device boot WiFi scan + periodic RSSI logging (`net.c`).
    Showed `FlyingSaucer` at −82 dBm (failing, NO_AP_FOUND); switched to `CommunityFibre10Gb_65587`
    (−80, same `192.168.1.x` LAN, reaches the gateway).
  - **Robotic/garbled audio (underrun on jittery WiFi):** gateway bursts ~1 s up front so the device
    holds a deep jitter buffer; device feeds **silence on underrun** (no stale-DMA buzz) and pins audio
    tasks to **core 1** (away from WiFi on core 0). User confirms audio now sounds fine.
- Device config (WiFi/gateway/token/user) is in git-ignored `firmware/main/secrets.h` (generated by `flash.sh`).
- Known follow-ups: −80 dBm is marginal (move closer / use `…00C08-ext` node for headroom); boot WiFi
  scan adds ~2 s to startup (remove for faster boot once stable); on-screen user picker still deferred.
